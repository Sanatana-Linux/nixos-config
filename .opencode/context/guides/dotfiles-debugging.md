# Accelerating Dotfiles Debugging

Core concept: Use `config.lib.file.mkOutOfStoreSymlink` to symlink dotfiles from the repo to their expected locations, bypassing Home Manager rebuilds for instantly-effective edits to non-Nix-managed config files.

Key Points:
- `mkOutOfStoreSymlink` creates symlink to absolute path outside Nix store
- Works for configs not generated by Nix (native neovim, awesome, etc.)
- Edit files in repo → changes take effect immediately, no `nixos-rebuild` needed
- This repo uses it for: `external/nvim` → `~/.config/nvim`, `external/awesome` → `~/.config/awesome`, `external/firefox` → Firefox chrome dir
- Only run `nixos-rebuild` once after adding new `mkOutOfStoreSymlink` declarations

Example:
```nix
xdg.configFile."nvim".source = config.lib.file.mkOutOfStoreSymlink /etc/nixos/external/nvim;
xdg.configFile."awesome".source = config.lib.file.mkOutOfStoreSymlink /etc/nixos/external/awesome;
```
